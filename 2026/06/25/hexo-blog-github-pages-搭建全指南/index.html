⚠️ 阅读前提：命令在哪儿执行？

本文所有命令都在 Windows 终端 中执行。打开方式：

  • cmd： Win + R → 输入 cmd → 回车
  • PowerShell： Win + R → 输入 powershell → 回车
  • Git Bash： Win + R → 输入 bash → 回车（需先安装 Git）

三种终端都能用，命令完全一样，选哪个都行。下文所有以 $ 或 > 开头的命令都在这类终端窗口中输入。

⚠️ 遇到的问题：social 链接 YAML 语法报错

配置社交链接时 YAML 解析报错：

原因： Butterfly 主题使用 || 作为分隔符，但 | 是 YAML 的块字面量标记。即使外层用了双引号，部分 YAML 解析器（Node.js 的 js-yaml）仍可能误解析。

解决方案： 用单引号包裹整个值，因为单引号内所有字符都按字面量处理：

教训： YAML 中遇到特殊字符（|、>、&、*、!、%、@、`、#、: 等），优先用单引号包裹整个字符串。

第七步：配置 Giscus 评论系统

Giscus 是一款基于 GitHub Discussions 的评论系统，免费、无广告、数据归你自己所有。

为什么选 Giscus？

评论系统 特点 是否免费
Giscus GitHub Discussions 驱动，数据在仓库里 ✅ 免费
Gitalk GitHub Issues 驱动，需创建 OAuth App ✅ 免费
Twikoo 腾讯云开发驱动，国内访问快 需云服务
Waline 功能全面，需单独部署后端 需服务器
Disqus 老牌评论系统 免费版有广告

对于已经用了 GitHub Pages 的博客，Giscus 是最省心的选择——不需要额外注册任何服务。

前置条件

  1. 开启 GitHub Discussions

    访问仓库 Settings 页面：https://github.com/<用户名>/<用户名>.github.io/settings

    向下滚动找到 Features 区域，勾选 Discussions。

    注意： Discussions 默认是关闭的，需要手动开启。

  2. 安装 Giscus App

    访问 https://github.com/apps/giscus → 点击 Install → 选择 Only select repositories → 勾选你的博客仓库 → 点击 Install 确认。

    Giscus App 是 GitHub 官方认证的开源应用，它只读取/写入你指定的仓库的 Discussions，不会碰其他仓库。

获取配置参数

  1. 打开 https://giscus.app/zh-CN
  2. 填入仓库名（如 hl-1/hl-1.github.io）
  3. 页面会自动识别出以下信息：
参数 含义 示例值
data-repo 仓库全名 hl-1/hl-1.github.io
data-repo-id 仓库唯一 ID R_kgDOTDtHFQ
data-category Discussions 分类 General
data-category-id 分类唯一 ID DIC_kwDOTDtHFc4C_12D

⚠️ 分类选择陷阱

Discussion 分类有权限区别：

分类类型 权限 适合评论吗
Announcements（公告） 仅管理员可发帖 ❌ 不适合
General（一般） 所有人可发帖/回复 ✅ 适合

务必选择 General，否则除了你自己，其他人无法评论。

总结

用 Hexo + GitHub Pages 搭建个人博客的核心流程：

  1. Init → npx hexo init blog
  2. Config → 改 _config.yml（站点名、URL）
  3. Theme → 装 Butterfly，改 _config.butterfly.yml
  4. Write → 在 source/_posts/ 写 .md 文章
  5. Preview → hexo server 本地看效果
  6. Deploy → hexo deploy 推上线

整个博客完全免费：内容托管在 GitHub Pages，无需服务器，无需域名费用。
