Hexo 个人博客迁移指南（换设备 / 重装系统）

概述

当你换了新电脑、重装系统，或者想在多台设备上维护同一个博客时，不需要从头搭建。Hexo 博客的迁移非常简单——只需复制源代码文件，在新设备上 npm install 即可恢复。

  • 适用场景： 换电脑、重装系统、多设备同步
  • 难度： 低
  • 耗时： 10 分钟

一、迁移原理

Hexo 博客由两类文件组成：

类型 文件 说明
源文件（需要迁移） 配置、文章、图片等 你手动写的，不可自动生成
生成文件（不需要迁移） node_modules、public、.deploy_git 执行命令自动生成

源文件通常只有几百 KB，打包很快。生成文件可能几百 MB，完全没必要搬。

打包方式

方式一：直接复制粘贴（推荐，最简单）

直接用 U 盘或网盘把 C:\workDoc\blog\ 整个文件夹拷过去。到了新设备后删掉 node_modules 和 public 即可。

方式二：用 Git（如果有自己的仓库）

在 C:\workDoc\blog\ 初始化 git 并推送到私有仓库，新设备 clone 下来。

总结

迁移的本质就三步：

  1. 打包源文件 → 几百 KB，几秒钟
  2. 新设备装 Node.js + Git → 一次性操作
  3. npm install + npm run deploy → 两条命令恢复

数据（文章评论、GitHub 仓库、部署历史）都在 GitHub 云端，不会因为换设备而丢失。
